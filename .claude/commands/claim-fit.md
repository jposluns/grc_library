Invoke the `claim-fit` skill defined in this project's pack at [`dev-security/claude-rules/skills/claim-fit/SKILL.md`](../../dev-security/claude-rules/skills/claim-fit/SKILL.md). Slash-command entry point is `/claim-fit`; the underlying skill name remains `claim-fit` (the descriptive name documents the workflow's purpose, the slash command is the ergonomic verb). This is the cadenced citation-precision audit of normative-attribution claims, catching the gate-blind FR-120 class, "attributed value, silent source" (a sentence ties a specific value to a named normative source, the source exists and the citation is well-formed so the existence and citation gates pass, but the source does not actually prescribe the attributed value). It is distinct from `/matrix-fit` (control-code semantic fit) and the content sweeps (`/validate`, `/validate-pr`): this judges whether cited sources actually state the values attributed to them. Run the one-time full Tier-A pass at adoption, then after any batch that adds or edits normative-value claims, and ad-hoc when a claim is in doubt. Execute the audit per the six-step process the skill encodes:

1. **Establish scope and confirm the reference base**: name the scope (the full Tier-A census for the adoption pass, the rows a just-applied batch introduced for the per-batch cadence, or a flagged set for the ad-hoc cadence). Confirm `tools/run_all_audits.sh` exits 0 first (a precision pass judges among claims that already pass the mechanical gates). Confirm the reference base is available: the `grc_library_ref` indexes (`grc_library_ref/INDEX.md`, `grc_library_ref/catalogue.yml`, `grc_library_ref/SECTION-INDEX.md`) locate the held source texts, and the per-source currency rule applies (confirm a held source is current before relying on it; a superseded held text is grounds to defer, never to judge against silently).

2. **Run the advisory triage tool to generate the worklist**: run `python3 tools/audit-claim-precision.py --ref-base <path-to-grc_library_ref-checkout>` (add `--tier A` for the Tier-A-only cadence). The tool always exits 0; its output is a recall-oriented worklist, tiered by risk (TIER A, a specific value tied to a named source in one clause, judged EVERY one; TIER B, soft-alignment phrasing with no specific value, sampled on cadence), with each named source family's held-state reported best-effort from the grc_library_ref indexes. Treat it as the judge's input-narrowing step, NOT a defect list. Add any claim the maintainer or a prior QA note flagged even if the extractor did not list it.

3. **Dispatch the citation-precision judge over the worklist**: dispatch one or more subagents (or read directly for a small worklist) to judge each claim. The brief: locate the named source's held text via the `grc_library_ref` indexes, read the specific clause, article, or annex the claim points at, and return one of the four verdicts (`prescribed` / `informed-not-prescribed` / `mis-attributed` / `source-not-held`) with the source passage QUOTED as evidence. The binding rules: judge against the held source text, never memory or plausibility; an un-held source is never adjudicated from recall, whatever the judge's confidence. Every judgement quotes the claim's `path:line`, the attributed value, the named source, and the source passage (or, for `source-not-held`, the index lookup that failed).

4. **Synthesize and apply-time-verify each candidate against the source text**: the orchestrator re-reads each candidate `mis-attributed` and `informed-not-prescribed` verdict's source passage in the reference base before treating it as a finding (the judge produces research; the orchestrator confirms). A judge false positive (a prescribing passage the judge missed elsewhere in the source, a version mismatch, an over-literal reading) is refuted here, not routed. Draft each confirmed fix per its verdict: a rephrased attribution for `informed-not-prescribed` (the value stands as the corpus's canonical choice; the sentence stops asserting source language that does not exist), a corrected value or citation for `mis-attributed` (surfaced to the maintainer when the choice is authorial), a source-drop queue entry for `source-not-held`. When an attribution is rephrased, grep the full touched file and the corpus for sibling carriers of the same value-plus-source pair at bare-token width.

5. **Triage and route findings**: confirmed in-scope findings are fixed in-window (apply the fix, bump the touched document's Version and Date in the same commit, record the correction in the CHANGELOG-detailed entry), with authorial choices surfaced to the maintainer as named options rather than silently picked. Confirmed out-of-scope findings are surfaced with named options (fix-now vs route-to-backlog), not auto-deferred. Findings refuted at apply-time are recorded with the refutation, not routed; findings that dedupe against an existing backlog item are cross-referenced, not duplicated.

6. **Record and surface**: surface confirmed findings inline in chat (per-finding: claim `path:line`, the attributed value, the named source, the verdict with the quoted source passage, the fix applied or the option surfaced). Write the run to the project's claim-fit record location ([`.working/claim-fit/`](../../.working/claim-fit/)) and append a history row; a zero-finding run still gets a history row (the proof-of-discipline), with no detail file. The pass terminates when the worklist is judged, the confirmed findings are routed or fixed, and the run is recorded; it is a single advisory pass, not a fix-to-fixed-point loop.

Reject any judge verdict that lacks an explicit claim `path:line` and a quoted source passage (or, for `source-not-held`, the failed index lookup). A verdict without quoted evidence is a hypothesis, not a finding; re-dispatch with a re-emphasized evidence requirement before synthesizing.

`claim-fit` does NOT replace the citation and existence gates (which run on every PR); it is the precision layer on top of them. A claim must pass the gates first; this skill judges precision among claims that already pass. A well-formed citation says nothing about whether the source states the attributed value.

Report back: the scope and mechanical baseline, the worklist the tool produced (Tier-A count, Tier-B sample, held-state summary), the judge's verdicts (per-claim: `path:line`, attributed value, named source, verdict, quoted passage), the apply-time verification outcomes (confirmed vs refuted), the fixes applied in-window and the items surfaced or routed, and the path to the run record.
