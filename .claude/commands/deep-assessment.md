Invoke the `deep-assessment` skill defined in this project's pack at [`dev-security/claude-rules/skills/deep-assessment/SKILL.md`](../../dev-security/claude-rules/skills/deep-assessment/SKILL.md). Slash-command entry point is `/deep-assessment`. This is the maintainer-invoked, rare-cadence whole-project deep assessment: it composes the existing semantic instruments (`/validate`, `/full-qa`, `/fitness`, `/matrix-fit`, `/claim-fit`, `/reference-audit`, `/screen-publications`, `/guardrails`) by invocation and adds the lenses the routine cadence does not apply to itself (gate-efficacy probing, blind-spot mapping, ground-truth citation sampling, adoptability and pipeline-integrity review, QA-ledger meta-audit). It is re-entrant: phase state lives in [`.working/deep-assessment/register.md`](../../.working/deep-assessment/register.md), so a bare invocation resumes an in-flight run at its next incomplete phase. It is NOT a substitute for the per-PR QA cadence and it is NOT `/trust-recovery` (which stays the reactive lapsed-confidence tier); it inherits that rule's routing and sign-off conventions with a proactive trigger. Execute per the eight-step process the skill encodes:

1. **Establish run state, environment integrity, and the live inventory**: read the register (resume if a run is `in-progress`, otherwise open a run row and a dated record file `.working/deep-assessment/YYYY-MM-DD-rN.md`); mechanically verify a full clone, all three repos, and the concurrency interlock; derive the live instrument inventory from the repo (the runners, the commands directory, the pack skills, the advisory tools, the specification's gate inventory), never from remembered counts, so the live inventory is the scope by construction and any quality-check instrument added later is covered automatically; record inventory and HEAD SHA.

2. **Confirm the mechanical baseline**: run both runners, the regression suite, both generator checks, and the sibling repos' validate gates, standalone and unpiped; all exit 0 before any semantic phase; record green-at-SHA and cross-check the handoff's asserted expectations.

3. **Run the project's semantic instruments, formally**: the pre-flight scanner then corpus-wide `/validate`; `/full-qa`; `/fitness`; whole-matrix `/matrix-fit`; Tier-A `/claim-fit` with a Tier-B sample; `/reference-audit` in FULL mode; `/screen-publications` over the reference base's `pending` publications; `/guardrails`; plus the advisory ledger aids. Full sanctioned shapes only, each with its own record; their findings also enter this run's routing. Where the harness supports per-dispatch model selection, run orchestration and finding-adjudication on the strongest available model tier, the wide fan-out readers on a cheaper tier, and the mechanical phases model-indifferent; invoked instruments' subagents inherit the session model, so set it deliberately at phase boundaries.

4. **Audit the audit programme itself**: the blind-spot map (`tools/audit-gate-blindspots.py`), the mutation probe (`tools/audit-gate-mutation.py`, DISPOSABLE copy only, never the working repos), the dead-gate and coverage analysis over full history, and an independent by-hand parity re-derivation across the four gate surfaces.

5. **Sample content accuracy against ground truth**: stratified citation sampling against the held reference-base texts via their indexes with per-source upstream currency confirmed this turn; per-domain adversarial coherence readers briefed to refute; unverifiable items labelled and tracked, never asserted.

6. **Assess adoptability, pipeline integrity, and the QA ledgers**: the fresh-adopter simulation from a bare clone; CI hardening and platform-verified branch protection plus a full-history secret and PII scan and guard/hook efficacy checks; the ledger meta-audit (sham-pass sampling, metrics trends, cross-repo reconcile).

7. **Verify, dedupe, tier, and route every finding**: orchestrator re-reads each cited source before routing; refutations recorded, not routed; confirmed findings deduped against the backlog and routed tiered by severity with none dropped; in-window mechanical fixes ship under the full per-PR QA cadence.

8. **Record, surface, and hold for maintainer sign-off**: write the run record and register update, surface the routed set tiered, and HOLD. The run terminates only on explicit maintainer sign-off; an empty finding set is presented for sign-off, never self-closed.

Reject any subagent or tool finding that lacks a source location and quoted or reproducible evidence; a verdict without evidence is a hypothesis, re-dispatched, not routed. Do not abbreviate any invoked instrument, do not hard-code inventories or counts, do not mutate the working repos, and do not run history-aware analysis on a shallow clone.

Report back: the run identifier and resumed-or-new status, the environment verification results and green-at-SHA baseline, the derived instrument inventory location, per-phase outcomes with each invoked instrument's record path, the findings register summary (confirmed / refuted / routed per tier, deduped cross-references), the register row state, and the explicit request for maintainer sign-off.
