Invoke the `reference-audit` skill defined in this project's pack at [`dev-security/claude-rules/skills/reference-audit/SKILL.md`](../../dev-security/claude-rules/skills/reference-audit/SKILL.md). Slash-command entry point is `/reference-audit`; the underlying skill name remains `reference-audit`. This is the cadenced reference-breadth audit between the corpus and the held reference base, in both directions, catching the gate-blind SP 800-154 class, "held but unused" (an authoritative source sits in the reference base while corpus documents its content would materially improve never engage it), and the reverse (a corpus document is touched while newly ingested or updated reference material that bears on it goes unnoticed). It is distinct from `/matrix-fit` (control-code fit), `/claim-fit` (claim precision), and the citation gates (existence and currency): this judges what a document SHOULD engage, not what it already cites. Run it in FULL mode as a `/deep-assessment` member and ad-hoc, in per-touch mode on every substantive corpus-document PR, and in new-ingest mode after reference-base changes. Trust tiers are load-bearing (maintainer decisions, 2026-07-08): standards, frameworks, legislation, and programs are authoritative (citation-grade improvements); templates are template-tier (content improvements, never normative citations); books are recommendation-tier only, never authoritative (corroborate against a trusted source before anything normative); publications are excluded by default. Execute the audit per the seven-step process the skill encodes:

1. **Establish scope, mode, and the reference baseline**: name the mode (FULL, per-touch with the touched document list, or new-ingest with the reference delta) and confirm `tools/run_all_audits.sh` exits 0 first (a breadth pass proposes additions to a corpus that already passes its gates). Confirm the reference base is available via its indexes (`grc_library_ref/INDEX.md`, `grc_library_ref/catalogue.yml`, `grc_library_ref/SECTION-INDEX.md`, `grc_library_ref/COVERAGE-MAP.md`); the per-source currency rule applies to any source a finding will rest on (confirm current upstream this turn before recommending engagement; a superseded held text routes a version-update item, never a silent recommendation).

2. **Run the advisory triage tool to generate the worklist**: run `python3 tools/audit-reference-breadth.py --ref-base <path-to-grc_library_ref-checkout>` in the mode's form (bare for FULL; `--docs <path> [<path> ...]` for per-touch; `--ref-since <sha>` or `--ref-items <substring>` for new-ingest). The tool always exits 0; its output is a recall-oriented worklist (per-item usage classification WELL-CITED / THIN / UNCITED / NO-KEY, plus topic-ranked candidate pairings), never a defect list. Non-book NO-KEY rows are alias-curation work for [`tools/reference-breadth-aliases.json`](../../tools/reference-breadth-aliases.json) in the same run; NO-KEY on books is expected. In per-touch mode, an empty candidate set for every touched document ends the run at this step with a one-line note in the PR's QA trail; no judge is dispatched on an empty set.

3. **Dispatch the reference-breadth judge over the worklist**: dispatch one or more subagents (or read directly for a small worklist) to judge each candidate pairing. The brief: read the corpus document in full, locate the held item's text via the reference-base indexes and read the relevant sections, and return one of the four verdicts (`adopt-citation` / `adopt-content` / `recommend` / `no-fit`) with the CONCRETE improvement named (which document section, engaging which part of the source, to what effect) and the source passage or structure quoted or pinpointed as evidence. Binding rules: judge against the held text and the live document, never a title, topic labels, or memory (a title-inferred verdict is the dominant judge failure mode); respect the tier ceiling (a book can never yield `adopt-citation`; a template yields `adopt-content`). In FULL mode, additionally give every UNCITED and THIN authoritative item an explicit disposition (genuine gap worth routing, or legitimately out of corpus scope with a one-line reason).

4. **Synthesize and apply-time-verify each candidate**: the orchestrator re-reads each `adopt-citation`, `adopt-content`, and `recommend` verdict's evidence (the source passage in the reference base and the named document section) before treating it as a finding. A judge false positive (the document already engages the source under different phrasing; the passage does not say what the verdict claims; a sibling document already covers it) is refuted here, not routed. Draft each confirmed improvement per its verdict and tier: citation-grade engagement for `adopt-citation`, content or structure adoption for `adopt-content`, and a corroboration-gated suggestion for `recommend` (name the trusted corroborating source or route the corroboration with the item).

5. **Triage and route findings**: small confirmed in-scope improvements are fixed in-window (apply, bump the touched document's Version and Date in the same commit, record in the CHANGELOG-detailed entry). Substantive improvements (a framework-alignment row set, a section rewrite, a multi-document engagement) are routed to TODO with evidence attached and surfaced to the maintainer with named options, never silently scheduled. FULL-mode structural outcomes route durably: under-used authoritative items to TODO; confirmed out-of-scope items recorded in the run record so the next run does not re-adjudicate them. Refutations are recorded, not routed; duplicates are cross-referenced against the existing backlog, not re-added.

6. **Update the per-document state**: in per-touch mode, after adjudication (or an empty set), run `python3 tools/audit-reference-breadth.py --docs <touched paths> --update-state` so [`.working/reference-audit/doc-state.md`](../../.working/reference-audit/doc-state.md) records the reference-base HEAD each document was audited against, and commit the state refresh with the PR's QA batch. In FULL mode, refresh the state for every document the run adjudicated. A document with no state row is treated as never audited and gets the full candidate set on its next touch.

7. **Record and surface**: surface confirmed findings inline in chat (per-finding: document path, held item and tier, verdict, concrete improvement, action taken or option surfaced). Write the run to [`.working/reference-audit/`](../../.working/reference-audit/) as a dated per-run record file (`YYYY-MM-DD-<scope>.md`, swept under the current-week model) and append a row to the non-dated `history.md`; a zero-finding or empty-candidate run still gets a history row (the proof-of-discipline), with no detail file. The pass terminates when the worklist is adjudicated, findings are applied or routed, the state is refreshed, and the run is recorded; it is a single advisory pass, not a fix-to-fixed-point loop.

Reject any judge verdict that lacks the concrete improvement and pinpointed evidence (source passage or structure plus document section), and any verdict that breaches its tier ceiling. A verdict without named evidence is a hypothesis, not a finding; re-dispatch with a re-emphasized evidence requirement before synthesizing.

`reference-audit` does NOT replace the citation gates or the sibling semantic audits; it is the breadth layer beside them. A confirmed `adopt-citation` finding lands as a normal corpus change that passes all of those gates, and any new normative value it introduces hands off to `/claim-fit`'s cadence.

Report back: the mode and scope with the mechanical baseline, the worklist the tool produced (classification counts, candidate counts, alias curation done), the judge's verdicts (per-pairing: document, held item and tier, verdict, evidence), the apply-time verification outcomes (confirmed vs refuted), the improvements applied in-window and the items routed with options, the state-file refresh, and the path to the run record.
