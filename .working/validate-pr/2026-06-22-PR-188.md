- **evidence:**

  Fitness-reviews README documents the filename convention as per-date:
  > "Filename `YYYY-MM-DD-rN.md` where `N` is the run ordinal on that date (default `r1`; increment if multiple runs in one day)."

  And history.md recorded it as r2.

- **impact:** The filename convention is per-date but the label conventions in the report internal narrative and in history.md were cumulative. The fitness-review skill spec also says "Multiple runs in one calendar day increment r1 → r2 → r3", which is also per-date. So the per-date interpretation is documented in two places; the cumulative interpretation in my drafted report and the history-row was my error.
