### Finding 1: High, IN-WINDOW: CHANGELOG over-claim on residual TLS 1.2

- **ruleId:** changelog-overclaim-residual-tls12
- **location:** `CHANGELOG.md:9` and `.working/changelog-details/CHANGELOG-detailed.md:35`
- **residual surface:** `operations/procedure-media-handling-and-transport.md:124`
- **evidence:** The #261 entry asserts "A corpus-wide contradiction search confirms no org surface still permits TLS 1.2" and "every org TLS-floor surface". But `operations/procedure-media-handling-and-transport.md:124` reads: "All digital transfers of Confidential or Restricted data ... must use an encrypted channel with TLS 1.3 or a higher-equivalent standard. **TLS 1.2 may be used only where a documented technical constraint prevents TLS 1.3**, with the constraint recorded in the exception register and reviewed quarterly." This is an org procedure that still permits 1.2 (via a governed exception).
- **root cause:** the contradiction-search regex was tuned to the *migrated* phrasings (`TLS 1.2 (minimum)`, `TLS 1.2+`, `Minimum version: TLS 1.2`); it did not match the "TLS 1.2 may be used" construction. The negative result was treated as corpus-complete (a `validate-inference-before-action` lapse: the bare token `TLS 1\.2` was the correct search, not a phrasing-specific regex).
- **rubric:** must-fix-this-window (CHANGELOG accuracy) + defer-surface-to-maintainer (the migration decision).
