## Purpose

This template captures the documentation accompanying any dataset used by the organisation for training, fine-tuning, evaluation, retrieval, or other AI processing. It is based on the datasheets-for-datasets pattern and is required for every dataset in the AI System Register. Each dataset has one datasheet; the datasheet evolves with the dataset and is versioned alongside it.

---

## Scope

This template applies to all datasets that influence model behaviour: training, fine-tuning, reinforcement-learning feedback, evaluation, retrieval-augmented generation knowledge bases, prompt example libraries, and synthetic-data generation seeds. It applies regardless of whether the dataset originated internally, was licensed from a supplier, was crawled or scraped under a documented basis, or was synthetically generated.

## Section 1: motivation

| Field | Required content |
| --- | --- |
| Dataset purpose | Why the dataset was created or acquired |
| Intended use cases | Specific models or systems this dataset supports |
| Out-of-scope uses | Uses for which the dataset must not be employed |
| Funding or commission | If the dataset was commissioned, by whom |
| Date created | Original creation date |
| Date acquired | Date the organisation acquired or generated this dataset |

## Section 2: composition

| Field | Required content |
| --- | --- |
| Modality | Text, image, audio, video, tabular, multimodal, embedding |
| Size | Number of records, total bytes, token count, hours of audio, etc. |
| Unit of record | What a single record represents (a document, a sentence pair, an image with caption, a user session) |
| Sampling method | Random, stratified, exhaustive, curated, scraped under defined rules |
| Subgroups | Demographic, geographic, language, topical subgroups present in the dataset |
| Imbalance assessment | Documented imbalance across subgroups |
| Labels | Whether records are labelled; label taxonomy; label provenance |
| Confidence in labels | Inter-annotator agreement or other quality metric |
| Missing data | Categories of records with missing fields and the imputation approach if any |
| Personal data | Whether the dataset contains personal data; if yes, identifiability tier per the pseudonymisation standard |
| Special-category data | Whether GDPR Article 9 categories or equivalents are present |
| Children's data | Whether subjects below the age of digital consent are represented |
| Confidential or restricted business data | Whether business-sensitive content is present |
| Synthetic content | If synthetic, the generation method and the seed dataset; whether outputs are labelled as synthetic |

| Field | Required content |
| --- | --- |
| Data sources | Identification of each source contributing records |

## Operating expectations

1. A datasheet is produced before a dataset is used in any AI training, fine-tuning, evaluation, or retrieval workflow.
2. Material change to the dataset (new sources merged, cleaning pipeline change, new labels added) triggers a datasheet update before the changed dataset is used.
3. Datasheets are linked from the AI System Register row for every system that consumes the dataset.
4. Datasheets for datasets containing personal data are reviewed by the Privacy Officer at minimum annually.
5. Where the dataset is acquired from a supplier, the supplier's own dataset documentation is appended as an annex.

## Limitations

This template is a public-domain baseline. Populated datasheets are dataset-specific and may contain commercially or legally sensitive information; populate, classify, and store internally. The template is not a substitute for the privacy impact assessment when one is required, nor a replacement for the model and system cards.
