## Purpose

This template structures the periodic risk report delivered to the board (or its designated risk-oversight committee). It expresses what the board should see, how it should be expressed, and how it should be cross-linked to source-of-truth registers, while remaining concise enough for board consumption.

The objective is informed challenge: the board can ask the right questions and form an opinion on the organization's risk posture, appetite, and treatment.

---

## Scope

The template applies to:

1. Regular (typically quarterly) risk reporting to the board or a board committee.
2. Ad-hoc reporting on material risk events between regular reports.
3. Annual statements that summarize the organization's risk position for governance purposes.

It does not replace specialist reports (information security, privacy, AI, financial risk) that have their own structures; it integrates and summarizes them.

## Report structure

### Section 1: cover

| Element | Description |
| --- | --- |
| Reporting period | The window the report covers |
| Date | The reporting date |
| Prepared by | The named risk officer responsible for content |
| Approved by | The named approver before submission to the board |
| Audience | Board, board risk committee, or equivalent |
| Confidentiality | Per the data classification standard |
| Distribution list | Recipients |
| Version | Report version |
| Linked source registers | Pointers to the underlying registers |

### Section 2: executive summary

A one-page summary stating:

| Element | Description |
| --- | --- |
| Risk posture statement | A one-sentence statement of overall posture (improving, stable, deteriorating) with rationale |
| Top three risks | The three risks that most warrant board attention this period |
| Material changes since last report | New, escalated, de-escalated, and closed risks |
| Key events in the period | Notable incidents, near-misses, regulatory actions, supplier events |
| Appetite breaches | Where actual exposure has exceeded stated appetite |
| Recommended discussion | Two or three questions the board is invited to consider |

### Section 3: risk appetite and tolerance

| Element | Description |
| --- | --- |
| Appetite restatement | One-line restatement per appetite domain |
| Position vs appetite | Heatmap or table comparing actual position against appetite per domain |
| Tolerance breaches | Where operational tolerances have been breached, with cause and treatment |
| Appetite revision recommendations | Where the executive recommends a change |

### Section 4: top enterprise risks

For each of the top enterprise risks (typically five to ten):

| Element | Description |
| --- | --- |
| Risk title | Short descriptive title |
| Risk class | Per the enterprise risk taxonomy |
| Inherent rating | Likelihood and impact combined |
| Residual rating | After existing controls |
| Trend | Improving, stable, deteriorating |
| Trajectory commentary | Why the rating is moving the way it is |
| Controls in place | Summary of existing controls; effectiveness rating |
| Treatment in progress | What the executive is doing about it |
| Treatment milestones | Key dates the board should know about |
| KRIs | Indicators monitoring this risk and their status |
| Risk owner | The named accountable executive |
| Linked enterprise risk identifier | Pointer to the enterprise risk register |

### Section 5: emerging risks

| Element | Description |
| --- | --- |
| Emerging risk | Short title |
| Source of signal | Horizon-scanning source |
| Plausibility | Tail, plausible, frequent |
| Possible impact | Range |
| Watching action | What the executive is doing to monitor or prepare |

### Section 6: incidents and near-misses

| Element | Description |
| --- | --- |
| Severity 1 and 2 incidents | List, with brief narrative and status |
| Near-misses of note | Material near-misses |
| Loss data | Financial and operational loss in the period |
| Customer impact | Customer-impacting incidents summarized |
| Regulatory reporting | Any regulatory notifications made in the period |
| Lessons learned | Cross-cutting lessons highlighted |

### Section 7: regulatory and external environment

| Element | Description |
| --- | --- |
| New or upcoming regulation | Material developments and the executive response |
| Regulatory engagement | Engagement with regulators in the period |
| Enforcement and inquiries | Any received in the period |
| External-environment signals | Market, geopolitical, technological signals affecting risk |

### Section 8: supplier and concentration risk

| Element | Description |
| --- | --- |
| Critical supplier position | Status of critical and important suppliers |
| Concentration risk | Summary from the concentration register |
| Fourth-party and beyond | Material visibility into deeper supply chains |
| Critical-ICT third-party regime | Where applicable, DORA / EBA equivalent |

### Section 9: AI and emerging-technology risk

| Element | Description |
| --- | --- |
| AI portfolio summary | High-level summary of AI use and risk classification |
| Foundation-model provider posture | Material provider events |
| AI incident summary | AI-specific incidents in the period |
| Regulatory AI position | EU AI Act readiness; sector AI regulation |

### Section 10: resilience and continuity

| Element | Description |
| --- | --- |
| Recovery posture | Service-recovery readiness summary |
| Exercise results | Tabletop and continuity exercise outcomes |
| Severe-but-plausible scenarios | Status against severe-but-plausible scenario testing |
| Material gaps | Gaps that warrant board attention |

### Section 11: assurance

| Element | Description |
| --- | --- |
| Audit coverage | Internal-audit coverage summary; per the assurance map |
| External audit | External-audit findings in the period |
| Regulator-driven assurance | Sector-specific assurance |
| Open audit findings | Material open findings; ageing summary |
| Independent-assurance opinion | Any independent attestations issued or received |

### Section 12: risk culture

| Element | Description |
| --- | --- |
| Speak-up signals | Whistleblower volume and theme |
| Risk-awareness training | Completion rates and themes |
| Conduct events | Material conduct-related events in the period |
| Culture-and-conduct survey signal | Where applicable |

### Section 13: board decisions requested

| Element | Description |
| --- | --- |
| Decisions sought | Specific decisions the board is asked to make |
| Items for note | Items the board is asked to note |
| Items for next time | Items being prepared for the next report |

### Section 14: appendices and supporting material

| Appendix | Description |
| --- | --- |
| Detailed enterprise risk register | Linked, not embedded |
| Detailed operational risk register | Linked |
| Loss data | Linked |
| KRI dashboard | Linked |
| Scenario test results | Linked |
| Assurance map | Linked |
| Glossary | If non-board-standard terms are used |

## Style and presentation

| Element | Expectation |
| --- | --- |
| Length | Executive-summary and core sections fit a board pack; full report kept proportionate |
| Plain English | Specialist jargon explained; first use defines acronyms |
| Visual summary | Heatmap, trajectory chart, top-risk dashboard suitable for board consumption |
| Forward-looking | Where appropriate, the report is forward-looking, not only backward-looking |
| Confidentiality | Sensitive content handled per classification; redactions noted, not silent |
| Consistency | Risk titles and identifiers stable across reports to enable comparison |

## Cadence

| Cadence | Audience | Content |
| --- | --- | --- |
| Quarterly | Board or board risk committee | Full template |
| Monthly | Enterprise Risk Committee (ERC) | Operational and material-change summary |
| Ad-hoc | Board chair, audit chair | Material event reporting |
| Annual | Board | Annual risk statement combining the four quarters |

---

## Preparation process

| Step | Description |
| --- | --- |
| Data collection | Risk officer aggregates from registers, incident logs, exercise results, audit findings |
| First draft | Drafted by the risk function |
| Executive review | Reviewed and adjusted by the Enterprise Risk Committee |
| Approver sign-off | Signed off by the Chief Risk Officer (or equivalent) |
| Board pack inclusion | Submitted per the board secretary's pack process |
| Verbal presentation | Risk officer presents and takes questions |
| Action capture | Board questions, requests, and decisions captured for follow-up |
| Follow-up | Actions tracked in the next report |

---

## Operating expectations

1. The same template is used each period to enable trend comparison.
2. Material changes are explained, not just stated.
3. The report is the source of board-visible risk reporting; ad-hoc reports reconcile to it.
4. The board's questions and requests are tracked across periods.
5. The report is reviewed annually for fitness; the board provides feedback on usefulness.
